# Contributing to awesome-claw

Thanks for helping improve the list.

## What belongs here

Resources that are specifically useful for the OpenClaw ecosystem, such as:
- official resources
- skills and skill tooling
- deployment and hosting paths
- memory and context systems
- clients, dashboards, and control surfaces
- channels and integrations
- reliability and maintenance tools
- community resources
- posts, threads, articles, demos, and showcases
- other curated OpenClaw lists

## Categories

When suggesting a resource, place it in the most relevant section if you know it:

- **Official Resources** — official docs, sites, and repositories
- **Skills** — skill collections, registries, and skill tooling
- **Deployment & Hosting** — installation and hosting paths
- **Memory & Context** — memory systems and context tooling
- **Clients, Dashboards & Control** — interfaces for using or managing OpenClaw
- **Channels & Integrations** — messaging and external service integrations
- **Reliability & Maintenance** — backup, monitoring, recovery, and maintenance tools
- **Communities** — community hubs and discussion spaces
- **Posts, Threads & Articles** — written and social resources worth reading
- **Showcases & Demos** — concrete examples of OpenClaw in action
- **More Lists** — other curated collections related to OpenClaw

## Submission guidelines

- Add items that are genuinely useful, not just tangentially related.
- Include a short note explaining why the item matters.
- If you know the best category, include it. If not, that's fine.
- Prefer stable, publicly accessible links.
- If the resource is early or rough, explain why it is still worth including.

## Quality bar

This is a curated list, not a directory.

Items may be rejected or removed if they are:
- stale
- low quality
- too promotional
- too loosely related to OpenClaw
- duplicates of existing entries

## Easiest way to contribute

Use the GitHub issue template for resource suggestions.

It now only asks for:
- the resource URL
- a short note on why it belongs
- optional category
- optional extra context

Keep it simple. We can sort out the formatting/details during review.
